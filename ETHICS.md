# Ethical Considerations — AI Study Coach

This document fulfils the ethical assessment requirement of the AI Engineering Capstone.

---

## 1. Data Privacy

### What data is collected
- **Conversation messages** — sent to the OpenAI API for processing. They are subject to OpenAI's data usage policy. Users should be aware of this before sharing sensitive personal information.
- **User ID** — a randomly generated string stored in browser `localStorage`. It contains no personally identifiable information (PII).
- **Study profile** — stored as a JSON file in `.study-memory/` on the server. Contains: topics studied, quiz scores, weak areas, and session count. No names, emails, or login credentials are stored.

### What data is NOT collected
- No real names, email addresses, or passwords
- No device fingerprinting or behavioural analytics
- No data is sold or shared with third parties

### Mitigations
- `OPENAI_API_KEY` is stored server-side in `.env.local` and never exposed to the browser
- `.env.local` is listed in `.gitignore` — it is never committed to version control
- Memory filenames are sanitised with a regex (`/[^a-zA-Z0-9_-]/g`) to prevent path traversal attacks
- For production deployments, the file-based memory store should be replaced with a properly secured database (e.g. Supabase with row-level security)

---

## 2. Bias and Fairness

### Sources of potential bias
- **Language model bias:** GPT-4o is trained on internet data that may contain cultural, gender, or socioeconomic biases. These can appear in quiz questions, explanations, or recommended topics.
- **Wikipedia grounding:** The `fetch_topic_summary` tool grounds responses in Wikipedia articles. Wikipedia has known coverage gaps — topics from non-Western or minority perspectives may be underrepresented or absent.
- **Difficulty inference:** The agent infers a user's preferred difficulty from average quiz scores. This could disadvantage users who are systematically tested on unfamiliar topics early in their usage.

### Mitigations
- The system prompt instructs the agent to adapt language to the student's apparent level, reducing the impact of assuming a single baseline
- Quiz questions are generated fresh each time, reducing the chance of a fixed, biased question bank
- The `fetch_topic_summary` fallback gracefully acknowledges when Wikipedia coverage is limited, rather than hallucinating
- Future improvement: periodically audit generated quiz questions for stereotyping or cultural bias

---

## 3. Academic Integrity

### Risk
An AI study coach could be misused — for example, a student might use it to generate answers to submit as their own work, or to take an open-book exam.

### Mitigations
- The system prompt explicitly positions the agent as a *learning* tool, not an answer service
- The agent is instructed to explain *why* an answer is correct, not just provide it — reinforcing understanding over copying
- The quiz flow hides correct answers until the student submits all responses, mirroring real exam conditions
- Users are responsible for complying with their institution's academic integrity policies; the tool does not verify context of use

---

## 4. Accuracy and Hallucination

### Risk
LLMs can generate plausible-sounding but factually incorrect information, which is especially harmful in an educational context.

### Mitigations
- The `fetch_topic_summary` tool fetches a Wikipedia summary *before* the agent explains any topic, grounding the response in a real, citable source
- When Wikipedia lookup fails, the agent explicitly acknowledges it is drawing from training knowledge rather than a verified source
- Quiz questions are generated by the LLM and are not verified against a curated question bank — users should treat them as practice material, not authoritative assessments
- Future improvement: implement a fact-checking step or link to primary sources for all explanations

---

## 5. Psychological Impact

### Risk
Negative feedback or repeated low scores could discourage learners, especially younger users.

### Mitigations
- The system prompt explicitly instructs the agent to be *encouraging* and to celebrate progress
- Topics with scores below 60% are labelled "Needs Review" (not "Failed") in the UI
- The agent is instructed to suggest next steps proactively, framing struggle as a normal part of learning
- The dashboard uses colour coding (green/amber/red) that is informative without being punitive

---

## 6. Over-reliance on AI

### Risk
Students may become dependent on the AI coach rather than developing independent study skills.

### Mitigations
- The `get_study_recommendations` tool produces plans the student is expected to execute themselves
- The agent does not do homework or write essays — it explains and tests
- The long-term memory system tracks mastered topics and adjusts difficulty upward, encouraging progression rather than repetition

---

## 7. Transparency

- The model being used (GPT-4o, GPT-4o Mini, or GPT-3.5 Turbo) is shown in the UI header at all times
- Token usage and estimated cost are displayed after each response
- The system prompt can be inspected in `lib/agent.js` — there are no hidden instructions that contradict the user-facing behaviour
